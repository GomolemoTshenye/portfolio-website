I'm Gomolemo Tshenye, a third-year IIE Bachelor of Computer and Information Sciences in Application Development student at The Independent Institute of Education (IIE) Emeris University in South Africa, and a developer who likes building systems that make a difference in the work space or public environment.

Before this degree I completed a IIE Higher Certificate in Mobile Application and Web Development , which gave me a grounding in both the front and back of app development and web development. The BCAD programme has been where the depth has come from I'm currently working through Programming and Information Systems modules that span full-stack development, Android, cloud integration,Web development and UX.

Most of my time outside class goes into projects. Wallet Willow is a cloud-synced personal-budgeting Android app I built in Kotlin with Firebase Auth, Firestore sync, Room for offline storage, and custom Canvas-based chart views I led the Firestore integration on a four-person team. TechLogistics is a logistics management system I refactored into a three-project .NET 10 solution (Web API with JWT auth, MVC frontend, NUnit tests) running on three named Docker containers over a bridge network. Emeris Pulse is a campus mobile app prototype I designed and built in React/TSX with Tailwind CSS, deliberately applying Interaction principles across eight screens.

On the side, I built CyberSecurity Chatbot is an enterprise-grade, full-stack conversational intelligence application built to assist IT administrators, developers, and security analysts. It streamlines threat identification, automates the parsing of system security logs, and provides immediate, actionable mitigation steps for common vulnerabilities. Instead of manually digging through massive security databases or complex log files, users can converse with the chatbot to diagnose system vulnerabilities and query threat intelligence in real time.

Alongside the technical work, I've spent time in administrative and customer-facing roles as a registration assistant supporting student enrollment and events, which has been genuinely useful for the collaboration and communication side of building software. I'm actively looking for opportunities to do this professionally in Tech and working environment.

What I care about: writing code that is correct, complete, and honest about its limitations; using version control like an adult; learning the new framework only when the old one stops solving the problem. Long term I'm aiming at full-stack engineering,Web development or a cloud-architecture path.
